## Purpose of the Code

This Python code defines an API endpoint for generating project documentation using a GPT-based language model (specifically Ollama). 

## Business rule

1. **User Input:**  Users submit their project directory path, preferred model, and system prompt through the POST request form.
2. **Documentation Generation:** The code uses `generate_documentation` to create textual documentation based on user input and a structured approach.
3. **Output:** The API sends back the generated documentation content in JSON format along with a success or error message if something went wrong.

## List external libraries


* **Flask:**  Used for creating an API framework.
* **swagger-flask_docs:** Generates Swagger UI documentation for user interaction. 
* **src/modules**: Contains modules for specific functionalities: 
    * `directory_structure`: Processes the project's directory structure based on the request data. 
    * `gpt`: Handles GPT model usage, likely for text generation.  
    * `file_processor`: Interacts with files (PDF, DOCX, TXT).


## fields

* **project_path:** (string) The path of the project directory provided in the request data. 
* **model:** (string) Name of the GPT model selected from a list of available models. 
* **gpt_provider:**  (string) Model provider name, default 'ollama'. 
* **selected_model:** (string) The user's selected model (obtained through request).
